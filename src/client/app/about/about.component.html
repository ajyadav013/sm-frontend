FrontEnd Features

  • Used Angular 4
  • Ready to go, statically typed build system using Gulp for working with TypeScript.
  • Production and development builds.
  • Development server with live reload.
  • TypeScript definition management using Types.
  • Basic Service Worker, which implements "Cache then network strategy".
  • MeView service so that only authenticated ones can view the home page else go to login page using router canActivate via AuthGuard

BackendEnd Features

  • Used Django
  • Knox login-logout functionality so as to access from multiple devices
  • Only frontend server allowed on allowed_hosts for backend accessibility
  • Need to add a local.py file for backend functionality for development purposes which includes the secret key and is a replica of local_template.py
  • Used SQLite database, I would prefer using Postgres with pgBouncer for connection pooling
  • Waitress can be configured to gunicorn workers or other servers with nginx configuration other than heroku, heroku itself provides load balancing with nginx
  • Installed Memcache and used locally, didnt setup for production. Can use different addons if used in heroku
  • Installed Celery but didnt use, can be used for sending email
